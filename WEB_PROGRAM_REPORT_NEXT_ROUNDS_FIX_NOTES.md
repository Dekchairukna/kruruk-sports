# WEB_PROGRAM_REPORT_NEXT_ROUNDS_FIX_NOTES

ปรับตามคำสั่งล่าสุด:

1. เอาข้อความ “เว้นว่าง = 0” ออกจากหน้าโปรแกรม/จัดการแข่งขันแล้ว
   - backend ยังรองรับการกรอกเร็วได้เหมือนเดิม แต่หน้าเว็บไม่โชว์ข้อความที่ทำให้สับสน
   - ผลที่ยังไม่บันทึกแสดงเป็น “รอผล” ไม่แสดงคะแนนหลอก

2. เพิ่มการแสดงเส้นทางรอบต่อไปล่วงหน้าในหน้าโปรแกรม
   - Round Robin/แบ่งกลุ่ม แสดงสูตรไขว้สายตั้งแต่ต้น เช่น
     - 2 กลุ่ม เอา 1-2: A1 พบ B2, B1 พบ A2
     - 4 กลุ่ม เอา 1-2: A1 พบ B2, B1 พบ A2, C1 พบ D2, D1 พบ C2
     - 4 กลุ่ม เอาเฉพาะที่ 1: A1 พบ D1, B1 พบ C1
     - 3 กลุ่ม + รองดีที่สุด 1: A1 พบ รองดีที่สุด, B1 พบ C1
   - ถ้าแข่งครบแล้ว ระบบจะแสดงชื่อทีมจริงในเส้นทางด้วย

3. เพิ่มการแสดงสาย Knockout ล่วงหน้าจนจบ
   - ไม่ต้องรอสร้างรอบในฐานข้อมูลก่อนก็เห็นว่า ผู้ชนะคู่ไหนจะไปพบผู้ชนะคู่ไหน
   - แสดงรอบต่อไป, ชิงชนะเลิศ และชิงอันดับ 3 (ถ้าเปิดใช้)

4. เพิ่มหน้าพิมพ์รายงานแบบเอกสาร
   - route ใหม่: `/events/<event_id>/program/print`
   - เปิดจากปุ่ม “พิมพ์รายงาน” ในหน้าโปรแกรม/จัดการแข่งขัน
   - มีโครงแบบรูปเล่ม:
     - หน้าปก
     - สารบัญ
     - ข้อมูลทั่วไปของงาน
     - ผู้บริหาร/ผู้รับผิดชอบจากข้อมูลในระบบ
     - ทีม/สี
     - โปรแกรมแข่งขันรายกีฬา
     - ตารางกลุ่ม
     - เส้นทางเข้ารอบ/สาย Knockout
     - กีฬาจัดอันดับ/กิจกรรมประกวด
     - ตารางเหรียญ/คะแนนรวม
   - ใช้ browser print / Save as PDF

5. เพิ่มปุ่มพิมพ์รายงานในหน้า Reports ด้วย

ตรวจแล้ว:

- Jinja template parse ผ่าน
- local default port ยังเป็น 5050
