# Web Competition Control Center

ปรับตามแนวคิด "แบบหน้าเว็บให้แอดมินใช้จัดการแข่งขันจริง" ไม่ใช่แค่สูจิบัตรหรือ PDF

## เพิ่มหน้าใหม่
- `/events/<event_id>/program`
- ชื่อหน้า: โปรแกรมและจัดการแข่งขัน
- รวมการทำงานในหน้าเดียว:
  - ดูรายการแข่งขันในอีเว้นท์
  - ดูโปรแกรมรายคู่
  - บันทึกผลแบบคะแนนรวมจากหน้าเดียว
  - เข้าใบบันทึกรายเซตสำหรับวอลเลย์บอล/ตะกร้อ/แบดมินตัน/เทเบิลเทนนิส
  - ดูตารางคะแนนกลุ่ม
  - กดสร้าง Knockout ต่อจากรอบแบ่งกลุ่ม
  - ดูสาย Knockout และกดตรวจ/สร้างรอบถัดไป
  - ดูรายการจัดอันดับและกิจกรรมประกวด

## ปรับหน้า Event
- เพิ่มปุ่ม `โปรแกรม/จัดแข่ง`
- การ์ดการแข่งขันในอีเว้นท์จะลิงก์เข้าหน้า `/program` พร้อม anchor ของรายการนั้น

## ปรับ Sidebar
- เพิ่มเมนู `โปรแกรมแข่ง`
- ยังเก็บ `สร้างการแข่งขัน` ไว้สำหรับสร้างรายการใหม่

## คะแนนว่าง = 0
- ปรับ backend ของ score-only match:
  - Round Robin score form
  - Knockout score form
- ถ้าช่องคะแนนว่าง ระบบบันทึกเป็น 0 เพื่อใช้งานหน้างานได้เร็ว
- รายเซตยังใช้ validation เดิมตามกติกากีฬา

## จัดรอบต่อ
- Round Robin: ถ้าบันทึกครบ จะแสดงปุ่มสร้าง Knockout ต่อในหน้าเดียว
- Knockout: เพิ่ม route `/knockout/<comp_id>/advance` สำหรับกดตรวจ/สร้างรอบถัดไป
- ระบบยังสร้างรอบถัดไปอัตโนมัติหลังบันทึกผลเหมือนเดิม

## ตรวจสอบ
- `python -m py_compile app.py models.py forms.py` ผ่าน
- Jinja parse สำหรับ `templates/events/program.html`, `templates/events/detail.html`, `templates/base.html` ผ่าน
- Local default port ยังเป็น 5050
